2. Подключаемся к telnet через Putty c помощью 80 порта, личного учебного сервера u69120.kubsu-dev.ru и протокола Raw (данные любого типа).

Протокол telnet, созданный на основе TCP, позволяет передавать на удаленный сервер простые текстовые команды для ручного управления процессами и включения взаимодействия между ними.

Запросы будем писать в текстовый файл и далее через буфер обмена мышью копировать запрос из текстового файла. Необходимо для сохранения синтаксиса и чтобы успеть прописать запрос до закрытия сессии.

3. Методом GET в протоколе HTTP 1.0 получаем главную страницу.

HTTP/1.1 200 OK - запрос выполнен успешно, ресурс найден.
Date: Tue, 18 Mar 2025 04:22:26 GMT - дата и время обработки запроса.
Server: Apache/2.4.58 (Ubuntu) - версия сервера, обрабатывающего запрос.
Last-Modified: Mon, 27 Jan 2025 15:24:32 GMT — когда страница была в последний раз изменена.
ETag: 29af-62b1c5a375e0 - уникальный идентификатор версии ресурса.
Accept-Ranges: bytes - указывает, что сервер поддерживает частичные запросы (range requests) для ресурсов.
Content-Length: 10671 - длина возвращаемого содержимого в байтах.
Vary: Accept-Encoding - указывает, что ответ может различаться в зависимости от заголовка Accept-Encoding в запросе. Connection: close - указывает, что соединение будет закрыто после передачи ответа.
Content-Type: text/html - указывает, что содержимое является HTML.
В ответ сервер выводит XHTML 1.0 Transitional документ с заголовком Apache2 Ubuntu Default Page: It works, подтверждающим, что сервер настроен должным образом.

4. Методом GET в протоколе HTTP 1.1 получаем внутреннюю страницу:

Ответ сервера сильно отличается от HTTP/1.0. Он более короткий, включает только успешно выполненный запрос, дату, версию сервера, длину и тип контента. Соединение открытое, это также отличает два протокола. В HTTP 1.0 соединение закрытое, устанавливаемое для 1 запроса, закрывается после каждого получения ответа на запрос. В HTTP 1.1 можно использовать одно соединение для нескольких запросов.

Примечание: при вводе запроса без Host: u69120.kubsu-dev.ru выводится ошибка 400 Bad Request.

Last-Modified: Mon, 10 Mar 2025 18:20:49 GMT - указывает на последний момент изменения указанного ресурса.
ETag: "c247-630010819142b" - уникальная метка версии ресурса, используемая для кэширования.
Accept-Ranges: bytes - указывает, что сервер поддерживает запросы диапазонов, позволяя извлекать части ресурса.
Content-Type: application/x-gzip - указывает, что возвращаемые данные сжаты в формате Gzip.
